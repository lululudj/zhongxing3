## 项目核心目标

**让老设备（GTX 1060 6GB + 16GB RAM）通过语义压缩跑大模型（如 DeepSeek V4-Flash）**

---

## 架构概览

```
用户输入 → 上下文-问题分离 → 语义压缩管道
                                     ↓
                              ┌─────────────────┐
                              │   三层智能体架构 │
                              ├─────────────────┤
                              │ 小脑：1.5B×3并行 │ ← 广度覆盖
                              │ 中脑：3B NER      │ ← 精度补充
                              │ 大脑：V4-Flash   │ ← 核心推理
                              └─────────────────┘
                                     ↓
                              紧凑格式 → 大脑推理 → 答案
```

## 压缩管道流程

1. **上下文-问题分离** — 用正则提取用户问题
2. **Schema发现** — 3B模型识别实体类型
3. **并行提取** — 1.5B×3提取实体/关系/属性
4. **RRF融合** — 多提取器结果合并去重
5. **补漏检查** — 检查遗漏实体
6. **共现扫描** — 降噪关系
7. **紧凑序列化** — 转换为机器语言格式

## 当前测试状态

### 最近测试结果
- **小说领域**：管道推理能正确回答"药老的叛徒弟子是谁？"
- **压缩比**：小说约3.4:1
- **瓶颈**：1.5B提取器对非叙事文本（法律/财报）效果较差

### 混合提取模式（已确定方案）
- 1.5B×3并行：保持广度覆盖
- 3B补充：不同时驻留，控制VRAM
- 目标：平衡准确率和显存占用
