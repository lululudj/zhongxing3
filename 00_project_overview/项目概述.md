# 项目概述

## 定位

**人语→机语：让老设备拥有长文本问答能力的语义压缩管道**

**最终目标硬件**：GTX 1060 6GB + 16GB RAM 级老旧设备。
**当前开发/测试硬件**：RTX 4060 Laptop 8GB + 16GB RAM（仅用于第一步验证，不是最终目标）。

通过轻量级 NLP 预压缩和结构化聚合，让 14B 模型在低配设备上获得长文本深度问答能力。

> **目标澄清 [2026-06-14]**：RTX 4060 Laptop 8GB 只是开发/测试机器。项目的真正目标是让 GTX 1060 6GB + 16GB RAM 的老旧设备也能通过语义压缩管道快速、准确、高效地跑大模型（如 DeepSeek V4-Flash）。在 4060 上跑通只是第一步验证，后续必须确保在 1060 这样只有 6GB VRAM 的低配设备上也能正常工作。所有性能推算、VRAM 管理策略、模型选型均以 GTX 1060 6GB 为最终约束条件。

## 核心理念

**压缩不是删减，是编码。**

人类语言充满冗余——修饰词、程度副词、因果连接词、等级模式等。这些对人类理解有帮助，但对模型推理是噪声。zhongxing 把"人语"转译成"机语"——零歧义的结构化紧凑表示，让小模型用最少的 token 完成推理。

## 核心洞察

**不是让小模型够用，而是通过压缩管道技术，让 256B 这种超大模型也能在老设备上快速稳定运行。**

输入极小，KV Cache 极轻，瓶颈从显存转移到语义保真度。

## v2.0 愿景升级

从"文本压缩器"升级为"模型调度器"——压缩 + 预判路由 + 难度标签 + 专家调度。

**三条破局路线：**
1. **预判路由** — 压缩时输出"专家预判表"，只需加载 20-50GB 而非 865GB
2. **深度可变推理** — 按问题难度动态调整推理层数（15-61层）
3. **蒸馏子网络** — 高频领域蒸馏到本地 14B，无需每次加载 V4-Pro

## 关键设计：无损压缩

这不是单向的有损压缩，而是含反馈环的无损管道：

| 机制 | 说明 |
|------|------|
| 语法压缩（有损语法，无损语义） | 去掉虚词/修饰词/冗余，实体/属性/关系/因果全部保留 |
| 语义保留（结构化提取） | 紧凑行格式保留全部关键信息 |
| 反馈兜底（feedback loop） | 大脑推理时信息不足 → 自动回溯原文补充 |

**丢掉的是冗余，不是信息。**

## 空 context 对照实验结论

- 14B 空 context 直接答 → 严重幻觉（编造不存在内容）
- 14B + 管道压缩 context → 所有答案在上下文范围内
- **准确率提升来自压缩检索，不是模型自带知识**
