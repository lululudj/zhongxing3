# 硬件环境与依赖

## 硬件体系（双层级）

zhongxing2 项目采用两阶段硬件策略：先在较优的开发机上验证可行性，再确保在真正的目标低配设备上也能正常工作。

### 开发/测试环境（当前可用）

| 部件 | 型号 |
|------|------|
| CPU | i5-12600K |
| GPU | **RTX 4060 Laptop 8GB** |
| RAM | 16GB DDR4 |
| SSD | NVMe |

> **角色**：仅用于第一步验证——确认语义压缩管道的可行性和正确性。**不是最终目标硬件。**

### 最终目标环境（待验证）

| 部件 | 型号 |
|------|------|
| CPU | 老旧 CPU（i5 级或更低） |
| GPU | **GTX 1060 6GB** |
| RAM | 16GB DDR4 |
| SSD | 任意 SSD / HDD |

> **这才是项目的真正目标**：让只有 6GB VRAM 的老旧设备也能通过语义压缩管道快速、准确、高效地运行大模型（如 DeepSeek V4-Flash）。

### 关键差异与设计约束

| 对比维度 | 开发机 (4060) | 目标机 (1060) | 影响 |
|----------|--------------|--------------|------|
| VRAM | 8GB | **6GB** | 目标机少 2GB，14B 模型 offload 更严重 |
| 14B 加载 | 刚好装下（9GB→部分 offload） | 必须大量 offload 到 RAM | 推理速度更慢，需更多优化 |
| 设计原则 | 验证正确性 | **验证可行性** | 4060 上跑通不保证 1060 能跑通 |

> **重要澄清 [2026-06-14]**：之前的文档曾错误地将 RTX 4060 Laptop 8GB 标记为"实际部署环境"，将 GTX 1060 6GB 标记为"原计划"。现已修正：**4060 只是开发测试机，1060 才是真正的最终目标。** 所有性能推算、VRAM 管理策略、模型选型均应优先考虑 GTX 1060 6GB 的约束条件。

## 软件依赖

- Python 3.12
- **Ollama 0.5.11** — 本地模型推理服务（运行在 127.0.0.1:11434）
- **无外部 Python 依赖** — zhongxing_agent.py 纯标准库（urllib + json + threading + re），不依赖 requests/pip

## VRAM 管理策略

Ollama 按需加载模型，1.5b 权重共享（3个并行提取不是同时驻留 3 份权重）。

### 开发机（RTX 4060 Laptop 8GB）
- 8GB VRAM 下 14B (~9GB) 会触发部分 GPU offload 到共享内存，推理速度可能受带宽影响
- 总内存占用约 15GB，勉强适配 16GB RAM

### 目标机（GTX 1060 6GB）
- 6GB VRAM 下 14B (~9GB) 必须大量 offload 到系统 RAM
- 推理速度将严重受限于 PCIe 带宽和 RAM 速度
- 可能需要进一步优化：考虑 7B-8B 级模型作为备选，或探索更激进的分层加载策略
- **这是项目的核心挑战所在——压缩管道必须足够高效，才能弥补硬件劣势**
