> **最高优先级 - 永久记录 - 不允许遗忘**

## 核心使命

**把不可能变为可能** —— 让老设备（RTX 3060 8GB + i5-10400F + 32GB DDR4）离线运行 **Qwen3-235B-A22B** 模型，15秒内回答

**目标模型：Qwen3-235B-A22B**
- 235B总参数 / 22B激活参数
- 128个专家，每次激活8个
- INT4量化后约117GB
- 需要 MoE混合推理 + mmap流式加载 + 专家卸载到CPU/SSD

---

## 目标模型：Qwen3-235B

### 模型选择理由

| 对比项 | Qwen3-235B | DeepSeek V3 |
|--------|------------|-------------|
| 总参数量 | 235B | 671B |
| 激活参数 | 预计更小 | 37B激活 |
| 内存适配性 | **更适合32GB内存** | 需要更大内存 |
| 模型类型 | MoE（混合专家） | MoE |

**选择结论：** Qwen3-235B比DeepSeek V3（671B/37B激活）更适合老设备，预计激活参数更小，更适合32GB内存约束。

### 模型来源
- **开发商：** 阿里通义千问3代
- **架构：** MoE（Mixture of Experts）
- **参数规模：** 235B总参数

## 技术路线

### 机语驱动循环管道（不是三层模型投票）

```
原文 → 1.5B小脑提取 → 3B纳米针压缩精炼 → 机语(300字)
                                              │
          ┌───────────────────────────────────┘
          ▼
    大脑读机语推理 → 答案 → 3B鹰眼标红圈 → 小脑重提取 → 循环
```

### 核心组件
- **1.5B小脑**：从原文粗糙提取实体/属性/关系 → 机语
- **3B纳米针**：机语压缩精炼 + 校验标注（不回答、不推理）
- **大脑**：只读精炼机语推理回答（**Qwen3-235B离线MoE**）
- **鹰眼**：红笔校验闭环（标问题→小脑重提取→大脑重答→循环）

### 关键理念
- **莫比乌斯环**：权重流式加载，无内外之分，循环处理
- **克莱因瓶**：机语和推理结果边界消失，闭环反馈

### MoE混合推理策略
| 组件 | 放置位置 | 原因 |
|------|----------|------|
| Attention层 | GPU | 常驻激活，需要高速 |
| 共享专家 | GPU | 每个token都用 |
| 路由专家(256个) | SSD + mmap | 按需加载8个 |
| 激活的8个专家 | CPU内存 | 当前计算需要 |

## 研究成果

1. **VRAM折叠**（莫比乌斯环理念）
2. **鹰眼校验闭环**（克莱因瓶理念）
3. **mmap流式加载**
4. **机语压缩**（原文3000字→机语300字，10:1压缩比）
5. **MoE混合推理**（GPU+CPU+SSD协同）
