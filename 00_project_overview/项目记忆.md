### [2026-06-15] 众星项目核心目的

**类别：** 核心目标 | **来源：** 用户定义 | **优先级：** 核心（最高）

> 本记录定义了众星项目存在的根本原因和最终目标，是所有技术决策的最高指导原则。

---

#### 1. 核心使命：把不可能变为可能

**终极目标：** 让普通设备离线运行超大模型（256B级别），实现15秒内回答

**核心洞察：**
- 传统认知：256B模型需要数百GB显存，普通设备无法运行
- 众星突破：通过机语压缩 + 管道架构 + MoE混合推理，让256B在8GB显存设备上稳定运行

---

#### 2. 核心技术路线（三层智能体架构）

| 层级 | 模型规格 | 核心职责 | 关键特性 |
|------|---------|---------|---------|
| **小脑** | 1.5B | 机语提取 | 从原文粗糙提取实体/属性/关系 |
| **中脑** | 3B | 机语精炼 | 压缩 + 校验标注（不回答、不推理） |
| **大脑** | 14B/256B | 推理回答 | 只读精炼机语进行推理 |
| **鹰眼** | 3B | 校验闭环 | 标红圈 → 小脑重提取 → 大脑重答 → 循环 |

**大脑选项：**
- 14B本地（GTX 1060级别）
- 256B云端（API调用）
- 256B离线MoE（RTX 4060 + CPU + SSD协作）

---

#### 3. 机语驱动的循环管道（莫比乌斯环 + 克莱因瓶）

**关键区别：** 这不是三层模型投票，而是机语流转管道

**设计理念：**

| 理念 | 核心思想 | 技术实现 |
|------|---------|---------|
| **莫比乌斯环** | 无内外之分，循环流式处理 | 机语在管道里流转、精炼、反馈、再提取 |
| **克莱因瓶** | 边界消失 | 机语和推理结果边界消失，无缝衔接 |

**流转过程：**
```
原文 → 小脑(提取) → 机语v1 → 中脑(精炼) → 机语v2 → 大脑(推理) → 答案
         ↑                                                      ↓
         ←←←←←←←←←← 鹰眼校验反馈 ←←←←←←←←←←←←←←←←←←←←←←←←←←←←←
```

---

#### 4. 硬件目标演进

| 阶段 | 硬件配置 | 目标 |
|------|---------|------|
| **原始目标** | GTX 1060 6GB | 跑14B模型 |
| **当前验证** | RTX 4060 8GB + i7-13620H + 32GB DDR5 | 离线跑256B MoE |
| **终极目标** | 任何设备 | 都能高效运行超大模型 |

**硬件约束意识：**
- RTX 4060只是开发验证环境
- 所有设计必须向下兼容到GTX 1060 6GB
- 最终目标：普及化、平民化

---

#### 5. 已完成研究成果

| 成果 | 描述 | 理念来源 |
|------|------|---------|
| **VRAM折叠** | 动态加载/释放，时序复用显存 | 莫比乌斯环理念 |
| **鹰眼校验闭环** | 轻量校验 + 反馈修正 | 克莱因瓶理念 |
| **mmap流式加载** | 零拷贝加载，减少内存占用 | 系统级优化 |
| **机语压缩** | 原文3000字 → 机语300字（10:1） | 语义编码 |
| **MoE混合推理** | GPU放Attention+共享专家，CPU/SSD放路由专家 | 异构计算 |

---

#### 6. 核心价值观

1. **不追求"够用"，追求"突破"** — 让小设备跑大模型，而不是让小模型在小设备上勉强工作
2. **无损压缩，不是有损删减** — 丢掉冗余，保留语义
3. **管道思维，不是投票思维** — 机语流转，层层精炼
4. **硬件普惠，不是硬件竞赛** — 让老设备焕发新生

### [2026-06-15] 核心设计理念

**类别：** 设计 | **来源：** 用户确认 | **优先级：** 核心

#### 1. 莫比乌斯环理念

**定义：** 模型权重循环流式处理

**流程：**
```
加载 → 推理 → 释放 → 循环
```

**核心思想：**
- 打破"大模型必须常驻显存"的传统思维
- 权重按需加载，用完即释放
- 通过循环复用有限的VRAM资源

**适用场景：**
- 多模型串行协作
- 有限显存环境（如GTX 1060 6GB）

---

#### 2. 克莱因瓶理念

**定义：** 边界消失设计

**核心思想：**
- KV cache与权重边界模糊化
- prefill输出直接作为下一层输入
- 消除传统架构中的边界瓶颈

**技术实现：**
- 跨层数据流优化
- 内存布局重组
- 避免中间结果的显式存储

---

#### 3. 鹰眼校验

**定义：** 老师"红笔模式"校验机制

**核心思想：**
- 只标问题，不修正
- 让推理模型自己决定如何修正
- 保持校验层轻量化

**工作流程：**
```
推理结果 → 鹰眼校验 → 标注问题位置 → 返回推理模型 → 自行修正
```

**优势：**
- 校验模型可以更小（如3B）
- 不需要完整的修正能力
- 减少校验层的VRAM占用

---

#### 4. VRAM折叠

**定义：** 动态管理模型生命周期

**核心原则：**
- 同一时刻只让一个大模型驻留GPU
- 模型用完立即释放
- 通过时序复用突破显存限制

**实现策略：**
```
时间片1: 小脑模型 (1.5B) → 加载 → 推理 → 释放
时间片2: 中脑模型 (3B)   → 加载 → 推理 → 释放
时间片3: 大脑模型 (14B)  → 加载 → 推理 → 释放
```

**约束条件：**
- 单个模型权重 < 可用VRAM
- 切换开销可接受
- 确保数据正确传递

---

#### 5. 机语格式

**定义：** 紧凑行格式结构化信息

**核心特征：**
- 零歧义的结构化表示
- 最小化token数量
- 机器可直接解析

**格式示例：**
```
E:实体名|T:类型|A:属性1=值1,属性2=值2
R:关系名|S:主体|O:客体|C:置信度
```

**优势：**
- 压缩比高（约3:1）
- 保留全部语义信息
- 模型推理效率提升
