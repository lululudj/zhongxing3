# 紧凑行格式

## 格式规范

```
[实体名] 属性1,属性2 | 关系1,关系2 | 因果1,因果2
```

### 字段说明
| 字段 | 含义 | 说明 |
|------|------|------|
| [实体名] | 实体名称 | 人物真名，如 [韩枫] |
| A 区 | 属性 | 短词，如 斗皇/厄难毒体/叛徒弟子 |
| R 区 | 关系 | 短词，如 师徒/朋友/敌人 |
| C 区 | 因果 | 用 → 连接因果链，如 偷袭→灵魂遁入骨灵冷火 |

## 示例

```
[韩枫] 叛徒弟子 | 师徒,弟子,师父 | 偷袭→灵魂遁入骨灵冷火
[药老] 师父 | 朋友,徒弟,不肖弟子,师父 | 灵魂体遁入骨灵冷火
[萧炎] 修炼师 | 徒弟,师傅,朋友 | 偷袭→灵魂遁入骨灵冷火
```

## vs JSON 对比

### 紧凑行格式（本方案）
```
[韩枫] 叛徒弟子 | 师徒,弟子,师父 | 偷袭→灵魂遁入骨灵冷火
```

### JSON 等价
```json
{"E":[{"n":"韩枫","A":["叛徒弟子"],"R":["师徒","弟子","师父"],"C":["偷袭→灵魂遁入骨灵冷火"]}]}
```

### 对比
| 指标 | JSON | 紧凑行 |
|------|------|--------|
| 格式开销 | ~30% | ~5% |
| 可读性 | 低（大量括号引号） | 高（表格化） |
| 14B 读取 | 需要解析 JSON | 直接读取 |
| 字符数 | ~120 | ~60 |

## 与晶体语言（v1.1）对比

v1.1 使用了更复杂的晶体语言协议（#M/#L0/#L1/#L2/#E/#R/#W），v1.5+ 简化为紧凑行格式，原因是：
- 更少的标记符号，降低模型解析负担
- 更紧凑的表示，减少 token 消耗
- 14b 可直接读取，无需额外解析层
