# 三层智能体架构

## 架构图

```
原文 → 分块 → [小脑×3 并行提取] → RRF融合+紧凑化 → 补漏+共现扫描 → 机语
                                                                          ↓
问题 → 大脑(14b) ← 机语(紧凑行格式) → 自然语言回答（含反馈环）
```

## 三层职责

| 层 | 模型 | 职责 | 实测大小 | 驻留内存 |
|---|------|------|---------|---------|
| 小脑 ×3 | qwen2.5:1.5b | 并行提取实体/属性/关系/因果 | 986MB | ~1GB |
| 中脑 | qwen2.5:3b | RRF融合(确定性) + NER补漏 + 维度自发现 | 1.9GB | ~3GB |
| 大脑 | qwen2.5:14b | 串行读取机语推理，自然语言回答（含反馈环） | 9.0GB | ~9GB |

> **架构确认 [2026-06-14]**：三Agent角色最终确认为 小脑(3×1.5b并行提取) → 中脑(3b RRF融合+NRE) → 大脑(14b串行读机语)。中脑的RRF融合是纯确定性算法（不调模型），NER补漏才用3b。

## 设计思路

把"塞不进显存"的问题转化为"并行小模型预处理 + 推理时只读精华"的工程方案：

- **小脑做广度** — 3 个 1.5b 同时跑，覆盖不同视角，防止单模型遗漏
- **中脑做精度** — 3b 在 NER 上比 1.5b 准，又比 14b 省资源
- **大脑做深度** — 14b 只读机语，不做提取，减少 token 消耗

## 并行 vs 串行

- **并行**：1.5b × 3 同时提取
- **串行链**：3b(维度发现) → 1.5b×3(并行提取) → 3b(补漏) → 14b(推理)

## VRAM 管理

Ollama 按需加载，1.5b 权重共享，不是三个独立模型同时驻留。

| 环境 | VRAM | 总内存占用 | 状态 |
|------|------|-----------|------|
| 开发机 (RTX 4060) | 8GB | ~15GB | 当前验证环境，14B 部分 offload |
| 目标机 (GTX 1060) | **6GB** | ~15GB | **最终目标**，14B 大量 offload，速度更严峻 |

> **关键约束**：三层架构的设计必须同时适配 6GB VRAM 的 GTX 1060。4060 上跑通只是第一步，1060 上的可行性才是真正的验证标准。
