# 开源小模型选型调研

> 调研时间：2026-06-13
> 目标：在 6GB VRAM 下寻找提取效果最佳的 3B 级模型

## 候选模型

| 模型 | 参数量 | 特点 | 评估 |
|------|--------|------|------|
| qwen2.5:1.5b | 1.5B | 当前使用，甜点区间下限 | ⭐ 可用 |
| qwen2.5:3b | 3B | 当前最佳提取器 | ⭐ 最优 |
| qwen3.5:4b | 4B | thinking 模式 + JSON 合规率低 | ❌ 死路 |
| gemma-3-4b | 4B | 待测试 | ❓ |
| phi-3.5-mini | 3.8B | 待测试 | ❓ |
| MiniCPM5-1B | 1B | 可替换 1.5b 小脑 | ❓ |

## 选型结论

### 提取器甜点区间：1.5B - 3B

**为什么 1.5B 是下限？**
- 太小信息密度不够
- 实体提取数量和质量均不足

**为什么 3B 是最优？**
- 实体提取数量和 JSON 合规率最佳
- VRAM 4105MB，在 6GB 范围内
- 耗时可控（67.9+37.7s）

**为什么 4B+ 不行？**
- JSON 合规率下降（思维链模式干扰）
- VRAM 超限（7641MB > 6GB）
- 5x 慢于 3B
- 伪高压缩比（实体太少）

### 中脑最佳模型

3b 在中脑任务上也表现最优：
- NER 准确率高于 1.5b
- 维度自发现更稳定
- 比 7b 更省资源

### 建议

**短期**：保持 qwen2.5:1.5b（小脑）+ qwen2.5:3b（中脑）组合
**中期**：测试 MiniCPM5-1B 替换 1.5b（更低显存，更高语义准确率）
**长期**：v2.0 路线下的模型调度器不受提取器限制
