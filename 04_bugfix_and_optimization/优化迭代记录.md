# 优化迭代记录

## 架构层面优化

### 1. 紧凑行格式替代 JSON（v1.5）
- **问题**：JSON 格式开销 30%，浪费 token
- **方案**：`[名] 属性 | 关系 | 因果` 格式
- **收益**：格式开销从 30% 降到 5%

### 2. 分块提取（v1.6.0）
- **问题**：1.5b 在长文本中提取崩塌，0 实体
- **方案**：chunk_text() 按段落边界分块 ~900 字
- **收益**：长文本提取恢复，正确率从 ~3/8 提到 ~4/8

### 3. 砍掉 3b 融合模型（v1.5）
- **问题**：3b 做融合会删实体、篡改事实、展开属性
- **方案**：全部改用确定性 RRF 融合
- **收益**：消除模型不确定性，提取结果可复现

### 4. 反馈环（v1.6.1）
- **问题**：单向压缩可能丢失信息
- **方案**：大脑输出 NEED_MORE → 原文补充 → 二次推理
- **收益**：信息缺失时有兜底机制

### 5. 分句共现窗口（v1.6.0）
- **问题**：大窗口共现扫描引入噪声
- **方案**：先按句号/换行切分，再按逗号/分号细分
- **收益**：缩小共现窗口，减少误匹配

## 算法层面优化

### 6. RRF 融合 + 模糊名去重
- 多路提取结果排序融合
- 模糊匹配合并（短名是长名子串则合并）
- 保留更短名（韩枫 > 韩枫是药老的弟子）

### 7. 子串去重
- 短属性是长属性子串 → 删短留长
- 例："叛徒" in "叛徒弟子" → 保留"叛徒弟子"

### 8. 0 实体自动重试
- 单次提取 0 实体 → 自动重试 1 次
- Pipeline 总实体 < 5 → 整体重跑

### 9. SPURIOUS_NAMES 黑名单
- 过滤 prompt 模板字面输出
- 配合最少 2 字名过滤

### 10. normalize_items 统一归一化
- str→list, dict→文本, 单字符过滤
- 防止字符拆解 bug

## 工程层面优化

### 11. 并行提取
- 3 个 1.5b 同时运行
- ThreadPoolExecutor 管理

### 12. Pipeline 级重试
- 重试仅在各提取器结果均不理想时触发
- 避免频繁无用重试

### 13. Ollama 按需加载
- 模型不在同一时间全部驻留
- 串行加载，先 3b 维度发现 → 1.5b 提取 → 3b 补漏 → 14b 推理

### 14. 零外部依赖 [2026-06-14]
- **问题**：原 requests 依赖在离线/干净环境需额外安装
- **方案**：zhongxing_agent.py 全部改用 urllib 标准库（urllib.request + urllib.error）
- **收益**：Python 3.x 开箱即用，无需 pip install 任何包
