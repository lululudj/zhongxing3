# Bug 修复记录

## v1.6.1 修复（2026-06-11）

### 1. 字符拆解 Bug

**现象**：1.5b 输出 `"C": "灵魂遁入骨灵冷火"`（字符串而非列表）时，Python 遍历字符串会逐字拆开，变成 `"灵" "魂" "遁" "入" ...`

**根因**：normalize_items 未处理 str 类型输入

**修复**：在 rrf_fuse/compactify_entity/serialize_compact 统一做 str→list 归一化

```python
def normalize_items(items) -> list:
    if isinstance(items, str):
        return [items] if len(items) >= 2 else []
```

### 2. 垃圾实体名

**现象**：1.5b 把 prompt 模板 `"n":"名"` 字面输出，导致出现"名"等垃圾实体

**根因**：1.5b 对 prompt 指令理解不精确，将 JSON 模板中的占位符当做实际内容

**修复**：SPURIOUS_NAMES 黑名单 + 最少 2 字名过滤

```python
SPURIOUS_NAMES = {"名", "的", "了", "是", "在", "和", "与", ...}
```

**修复**：子串去重——短属性是长属性子串则删短留长

```python
def substring_dedup(items: list) -> list:
    # "叛徒" in "叛徒弟子" → 删"叛徒"留"叛徒弟子"
```

### 4. Dict 值展平

**现象**：1.5b 有时在 R 字段输出嵌套 dict，如 `{'n': '冰皇'}`

**修复**：serialize_compact 检测 dict 类型并提取文本

## v1.6.0 修复（2026-06-11）

### 5. 分块提取崩塌

**现象**：1.5b 在 >2000 字文本中提取大量 0 实体

**根因**：小模型注意力窗口有限，长文本超出有效处理范围

**修复**：chunk_text() 自动按段落边界分块 ~900 字/块

### 6. 0 实体自动重试

**现象**：单次提取结果可能全空

**修复**：0 实体自动重试 1 次 + Pipeline 级重试（总实体<5 整体重跑）

### 7. 身份词噪声

**现象**：共现扫描将"女王/家主"等身份词当做关系

**修复**：去掉身份词关键词，只保留纯关系词

## v1.5.x 修复

### 8. 3b 融合模型问题（v1.5 砍掉）

**现象**：3b 做融合时会删实体、篡改事实、展开属性

**修复**：砍掉 3b 融合模型，全部改用确定性 RRF 融合 + 紧凑化

### 9. 共现关系丢失（v1.5.7）

**现象**：共现实体间的关系关键词（如"朋友"）未被提取

**修复**：确定性共现关系扫描，在分句窗口内检测关系关键词
